API Documentation

This document provides an overview of the technical requirements and implementation details for the Movie API.

1. Overview

The API is a Node.js and Express application designed to provide movie information and manage users.

2. REST Architecture

The API follows REST architecture principles, with the following endpoints:

  • GET /movies - Retrieves a list of all movies.
  • GET /movies/:id - Retrieves a single movie by its ID.
  • POST /users - Registers a new user.
  • PUT /users/:id - Updates user information.
  • DELETE /users/:id - Deletes a user by ID.

3. Middleware Modules

The API uses the following middleware modules:

  • body-parser - For parsing incoming request bodies.
  • morgan - For logging HTTP requests.
  • cors - To enable Cross-Origin Resource Sharing (CORS).

4. Package Management

The API uses a package.json file to manage dependencies. Example: 

{
    "name": "movie-api",
    "version": "1.0.0",
    "description": "A RESTful API for movie information.",
    "main": "index.js",
    "dependencies": {
        "express": "^4.17.1",
        "mongoose": "^5.10.9",
        "body-parser": "^1.19.0",
        "morgan": "^1.10.0",
        "cors": "^2.8.5"
    },
    "scripts": {
        "start": "node index.js"
    },
    "author": "Your Name",
    "license": "MIT"
}

5. Database

The API uses MongoDB as its database. The database schema and models are defined using Mongoose.

6. Movie Information

The API provides movie information in JSON format. Example response: 

{
    "title": "Inception",
    "director": "Christopher Nolan",
    "year": 2010,
    "genre": "Science Fiction"
}

7. Code Quality

The JavaScript code is thoroughly tested and is free of errors. Please use linters like ESLint to ensure code quality.

8. API Testing

The API is tested using Postman. Ensure that all endpoints work as expected, and responses are correct.

9. Authentication & Authorization

The API includes user authentication and authorization mechanisms. JWT (JSON Web Tokens) is used for securing the API endpoints.

Example of protected route:

app.get('/protected', authenticateJWT, (req, res) => {
    res.json({ message: 'This is a protected route' });
});

10. Data Validation

The API includes data validation logic to ensure that incoming data is correct and safe. Mongoose validators and middleware are used for this purpose.

11. Security

The API meets data security regulations. Sensitive data such as passwords are hashed using bcrypt, and all communication is done over HTTPS.

12. Source Code Management

The API source code is publicly accessible and maintained on GitHub. Ensure regular commits and proper versioning.

13. Deployment

The API is deployed to Heroku for public access. Ensure the app is set up to automatically deploy from the GitHub repository.
